# Markdown

## Что это?
Простой язык разметки, который превращает обычный текст в структурированный документ. Пишешь значки — получаешь заголовки, списки, жирный текст.

## Основной синтаксис (чего хватит на 95% случаев)

```
# Заголовок 1
## Заголовок 2

- Список
- Ещё пункт

**жирный** *курсив*

[ссылка](https://example.com)

![картинка](image.jpg)

`код в строке`
```

## Зачем реально нужен?

1. **Для README** — в каждом проекте на GitHub/GitLab
2. **Для документации** — в проектах, вики, базах знаний
3. **Для заметок** — в Obsidian, Notion, Typora
4. **Для сообщений** — в Slack, Telegram, Discord поддерживают
5. **Для блогов** — многие CMS понимают Markdown

## Почему лучше других форматов?

- **Проще Word** — нет танцев с форматированием
- **Легче HTML** — не нужны теги `<strong>`, `<ul>`, `<li>`
- **Текстовый формат** — открывается в любом редакторе
- **Версионный контроль** — Git отлично работает с `.md` файлами
- **Конвертируется во что угодно** → HTML, PDF, DOCX

## Реальный пример из жизни

Вместо этого в README:
```
УСТАНОВКА:
1. Скачать архив
2. Распаковать
3. Запустить setup.exe

ОСОБЕННОСТИ:
- Быстрый запуск
- Мало памяти
- Поддержка Windows 10+
```

Получается красиво отформатированная страница с заголовками и списками автоматически.

## Начать за 5 минут

1. Создай файл `readme.md`
2. Напиши что-то со `# Заголовок` и `- список`
3. Открой в любом современном редакторе (VS Code, Typora, онлайн-редактор)
4. Или загрузи на GitHub — увидишь готовый форматированный текст

**Итог:** Markdown — это как легкий HTML для людей. Не нужно учить теги, не нужно сложных редакторов. Написал `**текст**` — получил жирный текст. Идеально для документации, заметок и быстрого форматирования.
